# Catalyst是什么？

# 设计理念

Catalyst是基于模版技术的，数据驱动的后台管理系统页面快速搭建框架，采用vue技术开发，基于elementUI库做了二次封装，最低兼容Element-UI版本2.0.11。

支持搭建如下常见的管理系统页面：

  • 常规列表页
  • 表单页（待实现）
  • 工作台页（待实现）
  • 复杂详情页（待实现）

# 实现思路

基于组件化开发的思想，将典型的业务页面拆分成可复用的一系列组件，通过json格式的配置化数据控制组件的行为和页面的渲染交互，达到无需开发代码快速生成业务页面的目的。

# 资源管理

Catalyst建议所有的的资源（页面、组件、接口）都采用uri（统一资源定位符）来统一管理

# options机制

有的组件(下拉框、复选框)需要展示一些备选项，往往这些备选项是通用的数据。因此，Catalyst建议这类组件关联的备选项采用options机制来管理。在对应组件的配置项中通过optionsId属性来配置选项组的标识，调用注入的方法$mpGetOptions来获取备选项数据。

示例：

// 筛选条件组件的json配置
{
  'filters': [
    {
      'type': 'dropdown',
      'label': '账号类型',
      'key_name': 'acccount_type',
      'immediate': true,
      'options_id': 'account_type' // 此处配置的options_id，Catalyst会作为注入的$mpGetOptions方法参数传递
    },
  ]
}

// 筛选组件内部的逻辑
export default {
  ...
  inject: {
    $mpGetOptions: {
      default: () => null
    }
  },
  ...
  mounted() {
    this.init();
  },
  methods: {
    async init() {
      if (this.$mpGetOptions) {
        let options = await this.$mpGetOptions(this.optionsId);
      }
    }
  }
};

# 权限控制

对于页面级别的权限控制，需要特殊说明的是，页面上的tab切换交互场景。Catalyst将每一个tab页面认为是一个独立的页面，存在对应的URL，一组tab切换的组件就是若干页面的组合，这样的目的在于针对每个tab切换的页面实现独立的权限控制。

对于页面内部元素级别的权限控制，Catalyst目前是针对页面上的交互按钮可以配置相应的权限码(authority_code)，实现页面元素级别的精细控制。未来规划会扩展到页面上每一个元素都支持权限码的设置。
