# 搜索与筛选数据

# 搜索功能

搜索功能其实是一个特殊化的筛选组件，由一个下拉框，一个输入框和一个提交按钮组成，下拉框可以选择需要搜索的类型，输入框输入类型的模糊匹配。

为了展示并实现搜索功能，我们需要在filters中进行配置

filter: [
  {
    'uri': '/test',
    'type': 'keyword',
    'label': '关键字',
    'key_name': [
      'select_type',
      'select_content'
    ],
    'immediate': false,
    'options_id': 'keyword_fields'
  },
]

type filter类型，搜索组件的类型为keyword

label filter标签，这里是必填的，但在搜索组价中没有实际作用，可以随便填写，不为空就行。

key_name filter要传出的数据key，第一项为搜索类型对应的key，第二项为搜索内容对应的key

TIP

我们在 $mpSearch 函数中的param参数里可以取到在key_name里定义的数据，在这里附加到使用者的请求数据操作上。

# 搜索输入框的模糊匹配

搜索输入框可以进行模糊匹配，以快速定位我们要寻找的内容。 为了使用这个功能，开发者首先需要实现一个provider: $mpMatchKeyword，$mpMatchKeyword是一个async函数，其参数为uri和params。uri在搜索组件里配置，params参数的内容则与$mpSearch相同，为数据获取时的附加选项

$mpMatchKeyword: async (uri, params) => {
  return [
    {
      id: 0,
      value: '名称0'
    },{
      id: 1,
      value: '名称1'
    },{
      id: 2,
      value: '名称2'
    }
  ];
},

注意

uri必须被配置才能触发模糊匹配方法的调用

# 筛选器的配置

除了搜索组件，filter还可以接受更多更加通用的表单组件作为数据筛选，这些组件的配置会被置于搜索组件下方一个可收起展开的区域内。

参数 类型 默认值 说明
filters array [] 筛选器内的组件
<dh-sizer
  v-if="filters && filters.length > 0"
  :filters="filters"
  @sizerChange="handleSizerChange"
/>
<script>
  export default {
    data() {
      return {
        filters: []
      }
    },
    methods: {
      handleSizerChange(val) {
        // val是所有筛选器内子组件value的集合
      }
    }
  }
</script>

# 筛选器内组件的详细配置

# 通用配置

参数 类型 默认值 说明
type string 表单组件类型
label string 表单组件label
key_name string, array 表单组件值对应的key(数据上报的key)
immediate boolean false 该组件值改变时是否立即重新搜索数据
authority_code string 权限码
show_condition boolean, string 显示状态控制
grid number 0 当前组件占用几个栅格(最多24栅格)

# 筛选器组件分为以下几种

# text 文字输入框

普通的文字输入框

普通的输入框可以配置extension, extension里可以配置element-ui支持的属性

{
  'type': 'text',
  'label': '教师姓名',
  'key_name': 'name',
  'immediate': true,
  'extension': {
    'placeholder': '占位文本'
  }
},

# dropdown 下拉框

下拉框可以配置optionsId, 通过$mpGetOptions获取options

{
  'type': 'dropdown',
  'label': '账号类型',
  'key_name': 'acccount_type',
  'immediate': true,
  'options_id': 'account_type'
},

# calendar 日历

日历可以配置mode, 有以下四种模式

mode 说明
date 只选取日期
daterange 选取日期区间
datetime 选取日期时间
datetimerange 选取日期时间区间

当模式设置为区间时，key_name可接受数组，数组的两项分别代表开始时间和结束时间

{
  'mode': 'date',
  'type': 'calendar',
  'label': '日历(date)',
  'key_name': 'm_date',
  'immediate': true
},
{
  'mode': 'datetimerange',
  'type': 'calendar',
  'label': '日历(datetimerange)',
  'key_name': [
    'start_time',
    'end_time'
  ],
  'immediate': true
},

# radio 单选框组

可以配置optionsId, 通过$mpGetOptions获取options

{
  'type': 'radio',
  'label': '性别',
  'key_name': 'sex',
  'immediate': true,
  'options_id': 'sex'
},

# checkbox 复选框组

可以配置optionsId, 通过$mpGetOptions获取options

{
  'type': 'checkbox',
  'label': '城市',
  'key_name': 'city',
  'immediate': true,
  'options_id': 'city'
},

# switch 开关

switch开关可以配置extension, extension里可以配置element-ui支持的属性，目前支持active-text/inactive-text/active-color/inactive-color （参见element-ui）

{
  'type': 'switch',
  'label': '开关器',
  'key_name': 'switch_key',
  'immediate': true,
  'extension': {
    'active-text': '开启',
    'inactive-text': '关闭'
  }
},
