# 用户操作

需要用户交互操作的地方有两个，一个是用户交互区，位于筛选表单和表格之间，渲染出的操作会以按钮的形式显示在这里。另一个是表单中的interaction组件。渲染出的操作会绑定到interaction组件上

用户交互区的组件配置在actions.other参数中，而interaction组件的配置在field参数中，并与field的其他通用参数同级

# 通用配置

参数 类型 默认值 说明
mode string 交互类型
label string 交互按钮文本
uri string 页面链接或者api接口地址
batch string 是否批处理，mode: page不适用
key_name string, array 表单组件值对应的key
authority_code string 权限码
show_condition boolean, string 显示状态控制

# 打开外链

开启新的浏览器窗口打开外链，链接的地址由uri字段指定，不支持批量操作。如果该交互元素在数据表某行内，则外链会拼接上query参数entity_id=XXXX，参数值即为该行数据的entity_id

mode: 'page'

{
  'label': '查看外链',
  'mode':'page',
  'uri':'/detail'
},

# 发送请求

mode: 'ajax'

post请求访问接口，数据上报格式为application/json，接口地址由uri字段指定。如果该交互元素在数据表行内，post参数会传递该数据行的entity_id；如果在页面的操作区，则看是否批量勾选了数据行，将勾选的数据行(至少一行)的entity_id组装成一个数组上报。如果script字段配置了js脚本，则会执行script脚本的代码，将返回值作为数据上报。如果confirm_text字段有值，则在访问接口前先会弹出确认框提示用户，展示文字即为confirm_text字段值。

{
  'uri': '/api/mock/teacher/audit',
  'mode': 'ajax',
  'batch': true,
  'label': '审核',
  'script': '',
  'confirm_text': '确认通过吗？',
  'authority_code': '',
  'show_condition': ''
},

上报数据格式示例

// 单条数据
{
  entity_id: XXXX
}

// 多条数据
{
  entity_id: [
    XXXX,
    XXXX,
    XXXX,
    ......
  ]
}

// script字段传递了脚本字段，假设该脚本传递的函数返回值是{ a: 1, b: 'ok' }
{
  entity_id: [
    XXXX,
    XXXX,
    XXXX,
    ......
  ],
  a: 1,
  b: 'ok'
}

# 弹出对话框

mode: 'dialog'

在当前页面中部弹出层，层中展示的内容，通过uri指定的链接匹配。uri链接上会拼接query参数entity_id=XXXX，如果该交互元素在数据表内，则参数值只包含该行数据的entity_id；如果在页面的操作区，批量勾选了多行数据，则会传递多个entity_id以逗号分隔的拼接字符串

{
  'uri': '/plus/list/student',
  'mode': 'dialog',
  'batch': true,
  'label': '打开弹层',
  'authority_code': '',
  'show_condition': ''
}

uri链接参数格式示例

// 单条数据
'/XXXX/XXX/XXXX?entity_id=XXXX'
// 多条数据
'/XXXX/XXX/XXXX?entity_id=XXXX,XXXX,XXXX,XXXX'

# 弹出抽屉

mode: 'docker'

在当前页面从右往左滑出浮层，浮层内容，通过uri指定的链接匹配。uri链接上会拼接query参数entity_id=XXXX，如果该交互元素在数据表内，则参数值只包含该行数据的entity_id；如果在页面的操作区，批量勾选了多行数据，则会传递多个entity_id以逗号分隔的拼接字符串

{
  'uri': '/plus/list/student',
  'mode': 'docker',
  'batch': true,
  'label': '打开弹层',
  'authority_code': '',
  'show_condition': ''
}
