# 介绍

按照集成指南配置好列表模版页之后，就可以按照本指南配置不同页面的json配置数据，展示不同的业务页面。

# 它如何工作

Catalyst设计理念中，将管理系统列表页面划分为如下三个区域：

  • 筛选条件区 filters
  • 用户操作区 actions
  • 数据展示区 columns

当我们在JSON参数中配置好相关的内容，就可以自主生成一个包含搜索，筛选，排序，选择及分页等基本功能的列表页，并附带了常用的交互操作，比如弹框，弹层，ajax请求等。

# 配置概述

筛选条件区是若干表单组件的集合，数据展示区的每一列数据都会绑定一个或多个数据字段组件。

对于列表中的数据而言，Catalyst约定，每一行数据对应绑定的数据实体必须是扁平化的字段，并且要定义entity_id字段，用来唯一标识每一行数据。针对每条数据的操作或者批量的操作(例如审核、禁用等场景)，Catalyst会将entity_id字段上报到相关后台接口。

配置项的基本结构：

{
  filters: [],
  actions: {
      search: {},
      other: []
  },
  columns: []
}

一个基本的配置项示例如下：查看demo

{
  'actions': {
    'search': {
      'uri': '/api/mock/teacher/list',
    }
  },
  'columns': [
    {
      'field': {
        'type': 'raw',
        'key_name': 'id',
        'sortable': false
      },
      'label': '编号'
    },
    {
      'field': {
        'uri': 'http://www.dahai.com',
        'mode': 'page',
        'type': 'interaction',
        'key_name': 'name',
        'sortable': false,
        'show_condition': true
      },
      'label': '教师姓名'
    },
    {
      'field': {
        'type': 'raw',
        'key_name': 'age',
        'sortable': false,
        'show_condition': true
      },
      'label': '年龄'
    },
  ],
  'filters': [
    {
      'type': 'text',
      'label': '教师姓名',
      'key_name': 'name',
      'immediate': true
    },
  ]
}

同时我们暴露了接口以便于一些定制化的配置和扩展组件。
