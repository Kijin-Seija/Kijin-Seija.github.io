# 扩展指南

本指南面向需要扩展catalyst功能的前端开发者

Catalyst是基于模版技术的，数据驱动的后台管理系统页面快速搭建框架，采用vue技术开发，基于elementUI库做了二次封装。

通常可以认为，后台管理系统就是针对数据做一些操作处理，例如：增删改查等。每个页面我们规定，只针对一个数据实体进行操作，每个数据字段对应渲染一个展示/表单组件，通过双向绑定机制与用户交互关联起来。Catalyst已经预设了一批组件可以渲染不同交互，不同类型的组件，可以通过配置化的手段定制行为。当预设组件不满足业务需求是，开发者可以自行扩展组件，只需遵循本指南规定的扩展规范即可。

Catalyst目前支持扩展的组件分为两类：

  1. 表单组件，应用于列表页的筛选条件，表单页、工作台页面和详情页。
  2. 数据字段组件，应用于列表页的数据表格内部。

下面以表单组件为例，说明一下扩展组件流程。完整的示例可以参见Catalyst Demo工程。

# 第一步：开发自定义组件

每个组件实质上就是一个vue组件选项对象，只是需要满足一些属性和事件的要求。所以可以以任何方式开发、组织自定义组件，只要能导出vue组件选项对象即可。

  • 属性规范

    • 必须定义value属性，非表单组件value定义成普通prop即可，表单组件value需要实现双向绑定。value属性用来传递页面配置数据中的key_name绑定字段的值。value类型可以任意定义，如果key_name字段配置了多个绑定字段，则value定义成数组，一一对应返回数据。

    • 其它属性由开发者自己定义，没有强制要求，只是如果有其它属性，命名要求小驼峰命名。为了统一框架配置风格，有一些通用的组件属性字段建议开发者遵守：

      属性名称 类型 说明
      label String 组件显示的标题或者名称
      optionsId String 组件需要使用的选项组ID，通常是下拉框、复选框之类的组件会用到
      uri String 组件需要使用的资源ID，例如：调用的接口地址；引用的组件资源
      readOnly Boolean 组件是否只读状态

    • 每个组件的配置数据都会一一对应传递到组件的属性上，例如：配置中的options_id字段数据会传递到组件的optionsId属性上

      注意：页面配置数据中的字段我们采用下划线分隔的命名规范，而自定义组件中的属性我们采用小驼峰命名规范，框架在传递数据的时候会自动做两种命名规范的转换

  • 生命周期

    • 目前无特殊规则，开发者可以在vue组件支持的生命周期内做逻辑开发。

  • 事件规范

    • 自定义组件在渲染完成达到可用状态时必须触发catalystReady事件，用来通知catalyst框架该自定义组件已经初始化完成

  • options机制(可选) 有的组件(下拉框、复选框)需要展示一些备选项，往往这些备选项是通用的数据。因此，如果组件通过optionsId属性来获取备选项组的数据，需要调用注入的方法$mpGetOptions来获取。当然，自定义组件也可以不采用这个机制，而是内部通过自己的逻辑来获取备选项列表

    export default {
  ...
  inject: {
    $mpGetOptions: {
      default: () => null
    }
  },
  ...
  mounted() {
    this.init();
  },
  methods: {
    async init() {
      if (this.$mpGetOptions) {
        let options = await this.$mpGetOptions(this.optionsId);
      }
    }
  }
};

# 第二步：注册自定义组件

通过dh-catalyst的注册接口在运行时动态注册自定义组件

import Vue from 'vue';
import Catalyst from '@xes/dh-catalyst';
import '@xes/dh-catalyst/dist/catalyst.min.css';
import customComponent1 from 'custom-component1';
import customComponent2 from 'custom-component2';

Vue.use(Catalyst, {
  // 注册自定义表单组件，"XXXX"需要配置到模版中的type字段，这样框架才能知道渲染哪个组件
  formInputComponents: {
    'customType_1': customComponent1,
    'customType_2': customComponent2
  },
  // 注册自定义表格数据字段组件
  tableFieldComponents: {
    'XXXX': customComponent1,
    'XXXX': customComponent2
  }
});

经过前面的步骤，自定义组件就算是扩展完成可以使用了，下面说明如何配置自定义组件

# 配置自定义组件

在json配置中自定义组件必须配置的字段如下：

参数名称 类型 必填 默认值 说明
key_name String | String[] yes 绑定的表单字段名称，表单类的自定义组件需要与该字段配置的属性做双向绑定。
type String yes 自定义组件的类型，以第二步的例子而言，可以是“customType_1”

其它的字段诸如label、options_id等不是必须要实现的配置字段，但建议自定义组件按照配置指南的说明，沿用统一的字段命名。

Catalyst会将自定义组件的json配置数据一一对应传递到组件的属性上，自动进行小驼峰命名的转换。

# 配置示例如下：

{
  filters: [
    {
      label: '自定义表单组件名称1',
      type: 'customType_1',
      key_name: 'custome_field_name'
    }
  ]
}

# 一个完整的扩展表单组件的示例

# 编写组件代码

<template>
  <div>
    <span>{{ oLabel }}</span>
    <input
      :value="value"
      @input="handleInput"
    >
  </div>
</template>

<script>
export default {
  name: 'CascaderSelect',
  props: {
    value: {
      type: String,
      default: null
    },
    oLabel: {
      type: String,
      default: ''
    }
  },
  mounted() {
    this.$emit('catalystReady');
  },
  methods: {
    handleInput(e) {
      this.$emit('input', e.target.value);
    }
  }
};
</script>

# 注册组件

import Vue from 'vue';
import Catalyst from '@xes/dh-catalyst';
import '@xes/dh-catalyst/dist/catalyst.min.css';
import cascaderSelect from './components/cascader_select/index.vue';

Vue.use(Catalyst, {
  formInputComponents: {
    'cascaderSelect': cascaderSelect
  }
});

# 配置组件

{
  'filters': [
    {
      'type': 'cascaderSelect',
      'oLabel': '自定义筛选条件',
      'key_name': 'cus_field',
      'immediate': true
    }
  ]
}
