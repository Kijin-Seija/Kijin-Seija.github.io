# 集成列表页

# SDK接入

针对支持的每种页面，Catalyst都提供了一个相应页面的vue组件，来根据开发者配置好的json配置数据渲染页面。第三方系统引入sdk之后，即可在自己的页面上使用。 页面的json配置数据由第三方自行管理，只需满足Catalyst定义的配置规范即可。

Catalyst建议第三方开发者基于catalyst的SDK来开发模版页面，然后既可以编写满足catalyst配置规范的json配置数据，来生成满足不同需求的管理系统页面；当然开发者也可以基于正常的组件化开发流程，针对每一个业务页面，引用Catalyst组件独立开发

以常规列表页举例（以下涉及到的接口说明参见API Reference）

  1. 引入npm包

    // 安装
npm install --save @xes/dh-catalyst

    // 导入
import Vue from 'vue';
import Catalyst from '@xes/dh-catalyst';
import '@xes/dh-catalyst/dist/catalyst.min.css';

Vue.use(Catalyst);

  2. 实现权限接口（可选）

    Vue.use(Catalyst, {
  hasPermit(authorityCode) {
    return true;
  },
  getNameByUrl(url) {
    return 'url对应页面的名称';
  }
});

  3. 引入列表组件，配置组件属性，监听事件，实现slot，并实现相应的逻辑注入接口

    catalyst-list 是用于展示管理系统列表页面的组件。配置参数说明参见catalyst-list组件文档

    <template>
  <catalyst-list
    :page-title="pageTitle"
    :business-config="businessConfig"
    :initiate-condition="routeQuery"
    @dialogShow="handleDialogShow"
    @dockerShow="handleDockerShow"
  >
    <template #dialog>
      <!--页面如果有弹出居中弹层的交互，则此slot配置居中弹层的内容-->
      ......
    </template>
    <template #docker>
      <!--页面如果有右侧滑出浮层的交互，则此slot配置右滑浮层的内容-->
      ......
    </template>
  </catalyst-list>
</template>

<script>
export default {
  data() {
    return {
      pageTitle: '',// 页面标题
      businessConfig: null,// 页面配置模版数据，json格式
      routeQuery: {},// 页面路由query参数
    };
  },
  provide() {
    return {
      // 列表页关键字搜索的模糊匹配逻辑（可选）
      $mpMatchKeyword: async (url, params) => {
        ......

          {
            id: 0,
            value: '名称'
          },{
            id: 1,
            value: '名称'
          },{
            id: 2,
            value: '名称'
          }
        ];
      },
      // 列表页搜索逻辑，返回数据必须满足三个字段：list(当前页的数据条目列表)、total(数据条目总数量)、currentPage(当前页的索引)
      $mpSearch: async (url, params) => {
        ......
        return {
          list: [],
          total: 50,
          currentPage: 2
        };
      },
      // 列表页异步操作逻辑
      $mpDoAjaxAction: async (url, params) => {
        ......
      },
      // 页面上根据optionsId获取备选项逻辑
      $mpGetOptions: async (optionsId) => {
        ......
        return [
          {
            label: "合格",
            value: "1"
          },
          {
            label: "良好",
            value: "2"
          },
          {
            label: "优秀",
            value: "3"
          }
        ];
      }
    };
  },
  methods: {
    // 页面居中弹层显示的事件，参数arg有三个字段：uri(要显示内容的资源定位符uri)、confirm方法(触发弹层的确认处理，关闭弹层并重新刷新列表)、cancel方法(触发弹层的取消处理，关闭弹层)
    handleDialogShow(arg) {
      arg.uri;
      arg.confirm();
      arg.cancel();
    },

    }
  }

</script>
