# 介绍

Catalyst建议第三方开发者按照本集成指南文档来开发模版页面，然后按照配置指南编写满足catalyst配置规范的json配置数据，来生成满足不同需求的管理系统页面；

当然开发者也可以基于正常的组件化开发流程，针对每一个业务页面，引用Catalyst组件独立开发。

# 集成方式说明

第三方集成可以采取三种方式：

  • SDK方式，适用于第三方已有并且也是基于vue开发的管理系统集成Catalyst服务。
  • 微前端方式，适用于第三方已有的管理系统集成Catalyst服务，并且不要求第三方系统是基于vue开发。（待实现）
  • 独立系统方式，适用于第三方重新基于Catalyst服务搭建新的管理系统，配合可视化搭建平台可以快速搭建页面。（待实现）
