# catalyst-list组件文档

catalyst-list组件主要功能是将json格式的页面配置数据渲染成列表页面

# 用法示例

<template>
  <catalyst-list
    :page-title="pageTitle"
    :business-config="businessConfig"
    :initiate-condition="routeQuery"
    @dialogShow="handleDialogShow"
    @dockerShow="handleDockerShow"
  >
    <template #dialog>
      ......
    </template>
    <template #docker>
      ......
    </template>
  </catalyst-list>
</template>

<script>
export default {
  data() {
    return {
      pageTitle: '',
      businessConfig: null,
      routeQuery: {}
    };
  }，
  provide() {
    return {
      $mpMatchKeyword: async (url, params) => {
        ......
        return [
          {
            id: 0,
            value: '名称'
          },{
            id: 1,
            value: '名称'
          },{
            id: 2,
            value: '名称'
          }
        ];
      },
      $mpSearch: async (url, params) => {
        ......
        return {
          list: []
          total: 50,
          currentPage: 2
        };
      },
      $mpDoAjaxAction: async (url, params) => {
        ......
      },
      $mpGetOptions: async (optionsId) => {
        ......
        return [
          {
            label: "合格",
            value: "1"
          },
          {
            label: "良好",
            value: "2"
          },
          {
            label: "优秀",
            value: "3"
          }
        ];
      }
    };
  },
  methods: {
    handleDialogShow(arg) {
      arg.uri;
      arg.confirm();
      arg.cancel();
    },
    handleDockerShow(arg) {
    }
  }
};
</script>

# 组件属性

名称 类型 必需 默认值 描述
business-config Object true json页面配置数据
page-title String false '' 页面标题
initiate-condition Object false {} 初始的搜索条件

# 组件事件

# dialogShow

居中弹层显示事件

# dockerShow

右侧滑层显示事件

# 事件参数

Catalyst设计理念建议开发者使用uri(统一资源定位符)来管理vue组件资源，通过uri定位到某个vue组件

名称 类型 描述
uri String 弹层或滑层要显示内容的资源定位符uri
confirm function(): void 触发弹层或滑层的确认处理，关闭弹层并重新刷新列表
cancel function(): void 触发弹层或滑层的取消处理，关闭弹层

# 插槽

# dialog

dialog具名插槽用来显示居中弹层中的渲染内容

# docker

docker具名插槽用来显示右侧滑层中的渲染内容

# 依赖注入

catalyst-list组件一些具体逻辑功能是不适合抽象成通用逻辑的，例如：不同的系统可能使用不同的ajax请求库。因此这部分逻辑功能catalyst暴露出一些待注入的接口，由第三方引用catalyst-list组件的页面来注入实现逻辑。主要包括以下几个

方法名称 方法签名 必需 描述
$mpMatchKeyword async function(uri: String, params: Object): Array[{id: String|Number, value: String}] false 输入列表页关键字时，向后端发起模糊匹配的调用请求
$mpSearch async function(uri: String, params: Object): {list: Array, total: Number, currentPage: Number} true 列表页搜索逻辑，list中的数据项必须定义entity_id字段
$mpDoAjaxAction async function(uri: String, params: Object): void false 实现列表页操作区发起的异步操作逻辑，当页面配置了异步操作的交互按钮时，必须要实现此接口
$mpGetOptions async function(optionsId: String): Array[{label: String, value: String}] false 根据备选项组ID获取备选项数据，当页面的下拉框、复选框等配置了options_id字段时，必须要实现此接口，用来提供备选项

# $mpSearch方法说明

  • 参数说明

    名称 类型 必需 默认值 描述
    uri String true 在页面配置数据中配置好的搜索接口地址
    params Object true 搜索参数，包含页面上选择的筛选条件，另外还包含分页和排序字段：page_size（每页数据条目数量）、page（页码）、order_by（排序字段）、sort（排序顺序：'asc'或者'desc‘）

  • 返回数据说明

    名称 类型 必需 默认值 描述
    list Array[Object] true 当前页数据，list中每个数据实体必须定义entity_id字段，作为唯一标识
    total Number true 数据条目总数
    currentPage Number true 当前页码
